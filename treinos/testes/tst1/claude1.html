01

O Projeto da Prova

Fundamento

Antes de escrever uma linha de PHP, você precisa entender exatamente o que precisa entregar. A prova pede uma API REST — um conjunto de URLs (endpoints) que recebem e devolvem dados no formato JSON.

📋 O que é uma API REST?

Imagine um garçom em um restaurante. Você (o cliente) pede algo, o garçom (a API) vai à cozinha (o banco de dados) e traz o resultado. Você não vê a cozinha — só recebe o prato. No mundo web, a "cozinha" é o banco de dados MySQL e o "cardápio" são os endpoints.

🗂️ Endpoints que você precisará construir

MétodoURLFunçãoAuth?
POST/api/registroCadastrar usuárioNão
POST/api/loginFazer login → gera tokenNão
GET/api/logoutFazer logout → apaga tokenSim
POST/api/banco_questoesAdmin: cadastrar questãoAdmin
POST/api/provaCriar prova personalizadaSim
POST/api/prova/{id}/responder/{id_pergunta}Responder questãoSim

POST/api/prova/{id}Consultar provaSim
GET/api/metricasVer desempenho do usuárioSim

🗃️ Banco de Dados — Estrutura necessária

Com base nos endpoints, você precisará das seguintes tabelas no MySQL:

-- Tabela de usuários
users: id, nome, username (único), senha (hash), token, is_admin

-- Tabela de questões do banco
questoes: id, enunciado, dificuldade, materia, created_at

-- Tabela de opções de cada questão
opcoes: id, questao_id, letra, opcao, opcao_correta (boolean)

-- Tabela de provas criadas
provas: id, user_id, dificuldade, quantidade_questoes,
         materias (JSON), finalizada, created_at

-- Pivot: questões associadas a cada prova
prova_questoes: id, prova_id, questao_id

-- Respostas dadas pelo usuário
respostas: id, prova_id, questao_id, user_id, resposta_dada
💡 Dica estratégica: Sempre que não souber o que fazer durante a prova, volte para este mapa de endpoints. Cada endpoint é um "mini-projeto" independente. Entregue os mais simples primeiro (registro, login) para garantir pontos.

PHP Básico — Do Zero

Fundamento
Contexto: Como você já conhece lógica de programação, vamos focar nas particularidades do PHP que diferem de outras linguagens, sempre mostrando como elas serão usadas no projeto.

// Variáveis começam com $ (obrigatório!)

🌐 Recebendo Dados HTTP (crucial para a API!)

<?php

// Ler o método HTTP da requisição
$metodo = $_SERVER['REQUEST_METHOD']; // "GET", "POST", etc.

// Ler body JSON (como será na API)
$body = file_get_contents('php://input');
$dados = json_decode($body, true);

$nome     = $dados['nome'] ?? null;
$username = $dados['username'] ?? null;
$senha    = $dados['senha'] ?? null;

// Ler headers (para pegar o token de autenticação)
$headers  = getallheaders();
$token    = $headers['Authorization'] ?? null;

// Devolver resposta JSON
header('Content-Type: application/json');
http_response_code(200); // ou 401, 422, 404...
echo json_encode(['mensagem' => 'Cadastro realizado com sucesso']);

?>
⚠️ Atenção: O operador ?? (null coalescing) é seu melhor amigo na API. Ele retorna o valor se existir, ou null se não existir — evita erros quando um campo não é enviado.

PHP + MySQL com PDO

Prático
Por que PDO? PDO (PHP Data Objects) é a forma segura e moderna de conectar PHP ao MySQL. Ela previne ataques de SQL Injection automaticamente com prepared statements. É o padrão que você deve usar na competição.

🔌 Conectando ao Banco

<?php

// config/database.php
function conectar() {
  $host   = 'localhost';
  $banco  = 'enem_db';
  $user   = 'root';
  $pass   = ''; // vazio no XAMPP local

  try {
    $pdo = new PDO(
      "mysql:host=$host;dbname=$banco;charset=utf8",
      $user, $pass
    );
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    return $pdo;
  } catch (PDOException $e) {
    http_response_code(500);
    echo json_encode(['mensagem' => 'Erro de conexão']);
    exit();
  }
}

?>

📝 CRUD Completo

<?php

$pdo = conectar();

// ─── INSERT (Cadastrar usuário) ─────────────────
$stmt = $pdo->prepare(
  "INSERT INTO users (nome, username, senha) VALUES (?, ?, ?)"
);
$senhaHash = password_hash($senha, PASSWORD_BCRYPT);
$stmt->execute([$nome, $username, $senhaHash]);


// ─── SELECT (Buscar usuário pelo username) ──────
$stmt = $pdo->prepare(
  "SELECT * FROM users WHERE username = ?"
);
$stmt->execute([$username]);
$usuario = $stmt->fetch(PDO::FETCH_ASSOC);
// $usuario agora é um array: ['id'=>1, 'nome'=>'João', ...]


// ─── SELECT múltiplos (listar questões) ─────────
$stmt = $pdo->prepare(
  "SELECT * FROM questoes WHERE dificuldade = ? AND materia = ?"
);
$stmt->execute([$dificuldade, $materia]);
$questoes = $stmt->fetchAll(PDO::FETCH_ASSOC);
// $questoes é um array de arrays


// ─── UPDATE (salvar token no login) ─────────────
$token = md5($username);
$stmt = $pdo->prepare(
  "UPDATE users SET token = ? WHERE id = ?"
);
$stmt->execute([$token, $usuario['id']]);


// ─── DELETE (logout — apagar token) ─────────────
$stmt = $pdo->prepare(
  "UPDATE users SET token = NULL WHERE token = ?"
);
$stmt->execute([$token]);

?>

🗺️ O Roteador — api/index.php

Este arquivo recebe todas as requisições e decide qual arquivo executar:

<?php

header('Content-Type: application/json');
header('Access-Control-Allow-Origin: *');

$uri    = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH);
$metodo = $_SERVER['REQUEST_METHOD'];

// Remove o prefixo do projeto da URL
$uri = str_replace('/meu_projeto/api', '', $uri);

// Roteamento
if ($uri === '/registro' && $metodo === 'POST') {
  require 'registro.php';

} elseif ($uri === '/login' && $metodo === 'POST') {
  require 'login.php';

} elseif ($uri === '/logout' && $metodo === 'GET') {
  require 'logout.php';

} else {
  http_response_code(404);
  echo json_encode(['mensagem' => 'Rota não encontrada']);
}

?>

<?php

require_once '../config/database.php';

// 1. Ler os dados enviados

  exit();
}

// 4. Criar usuário
$hash = password_hash($senha, PASSWORD_BCRYPT);
$stmt = $pdo->prepare(
  "INSERT INTO users (nome, username, senha) VALUES (?, ?, ?)"

$stmt->execute([$nome, $username, $hash]);

// 5. Retornar sucesso
http_response_code(200);

Autenticação por Token

Prático

A prova exige que todas as rotas (exceto registro e login) sejam protegidas por token. Entender esse fluxo é fundamental.

🔐 Como funciona o fluxo de autenticação

1. Usuário faz POST /login com username + senha
   → API verifica no banco
   → Gera token MD5 do username
   → Salva token no banco
   → Retorna: { "token": "abc123..." }

2. Nas próximas requisições, o usuário envia:
   Header:  Authorization: abc123...

3. A API verifica se esse token existe no banco
   → Encontrou? Usuário válido, prosseguir
   → Não encontrou? Retornar 401 "Usuário inválido"

🛡️ Helper de autenticação — helpers/auth.php

<?php

function autenticar($pdo) {
  $headers = getallheaders();
  $token   = $headers['Authorization'] ?? null;

  if (!$token) {
    http_response_code(401);
    echo json_encode(['mensagem' => 'Usuário inválido']);
    exit();
  }

  $stmt = $pdo->prepare("SELECT * FROM users WHERE token = ?");
  $stmt->execute([$token]);
  $usuario = $stmt->fetch(PDO::FETCH_ASSOC);

  if (!$usuario) {
    http_response_code(401);
    echo json_encode(['mensagem' => 'Usuário inválido']);
    exit();
  }

  return $usuario; // Retorna dados do usuário logado
}

function autenticarAdmin($pdo) {
  $usuario = autenticar($pdo);

  if (!$usuario['is_admin']) {
    http_response_code(401);
    echo json_encode(['mensagem' => 'Usuário inválido']);
    exit();
  }

  return $usuario;
}

?>

✅ Usando a autenticação em uma rota protegida

<?php

// qualquer_rota_protegida.php
require_once '../config/database.php';
require_once '../helpers/auth.php';

$pdo     = conectar();

// Se chegou aqui, $usuario está autenticado
echo json_encode(['mensagem' => 'Olá, ' . $usuario['nome']]);

?>

🎮 Controller — exemplo do AuthController

# Criar controller
php artisan make:controller AuthController
<?php
namespace App\Http\Controllers;
use App\Models\User;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Hash;

class AuthController extends Controller {

  public function registro(Request $request) {
    // Validar dados
    if (!$request->nome || !$request->username || !$request->senha) {
      return response()->json(
        ['mensagem' => 'Erro ao cadastrar, verifique os dados'], 422
      );
    }

    // Verificar username único
    if (User::where('username', $request->username)->exists()) {
      return response()->json(
        ['mensagem' => 'Erro ao cadastrar, verifique os dados'], 422
      );
    }

    User::create([
      'nome'     => $request->nome,
      'username' => $request->username,
      'senha'    => Hash::make($request->senha),
    ]);

    return response()->json(['mensagem' => 'Cadastro realizado com sucesso']);
  }

  public function login(Request $request) {
    $usuario = User::where('username', $request->username)->first();

    if (!$usuario || !Hash::check($request->senha, $usuario->senha)) {
      return response()->json(['mensagem' => 'Dados incorretos'], 401);
    }

    $token = md5($usuario->username);
    $usuario->update(['token' => $token]);

    return response()->json(['token' => $token]);
  }
}
?>

Construindo a API Completa

Avançado

🏗️ Criação de Prova — Lógica mais complexa

Este é o endpoint mais desafiador: você recebe matérias e dificuldade, busca questões aleatórias e cria uma prova:

<?php
// ProvaController.php

public function criar(Request $request) {
  $usuario   = $request->user(); // usuário autenticado
  $dificuldade  = $request->dificuldade;
  $quantidade   = $request->quantidade_questoes;
  $materias     = $request->materias; // array de strings

  // Validar campos obrigatórios
  if (!$dificuldade || !$quantidade || !$materias) {
    return response()->json(['mensagem' => 'Dados inválidos'], 422);
  }

  // Buscar questões aleatórias do banco
  $questoes = Questao::where('dificuldade', $dificuldade)
    ->whereIn('materia', $materias)
    ->inRandomOrder()
    ->limit($quantidade)
    ->get();

  // Criar a prova
  $prova = Prova::create([
    'user_id'           => $usuario->id,
    'dificuldade'        => $dificuldade,
    'quantidade_questoes'=> $quantidade,
    'materias'           => json_encode($materias),
    'finalizada'         => false,
  ]);

  // Associar questões à prova
  $prova->questoes()->attach($questoes->pluck('id'));

  return response()->json([
    'id'                 => $prova->id,
    'perguntas'          => $questoes->load('opcoes'),
    'finalizada'         => false,
    'dificuldade'        => $dificuldade,
    'quantidade_questoes'=> $quantidade,
    'materias'           => $materias,
    'created_at'         => $prova->created_at,
  ]);
}
?>

📊 Métricas do Usuário — Lógica de aggregação

<?php

public function metricas(Request $request) {
  $userId = $request->user()->id;

  // Total de provas
  $totalProvas = Prova::where('user_id', $userId)->count();

  // Buscar todas as respostas com a questão associada
  $respostas = Resposta::where('user_id', $userId)
    ->with('questao')->get();

  $corretas   = 0;
  $erradas    = 0;
  $materias   = ['portugues'=>0, 'matematica'=>0,
                  'biologia'=>0, 'fisica'=>0, 'quimica'=>0];
  $materiasErradas = $materias;

  foreach ($respostas as $r) {
    $opcaoCorreta = $r->questao->opcoes
      ->where('opcao_correta', true)->first();

    $mat = strtolower($r->questao->materia);

    if ($opcaoCorreta && $r->resposta_dada === $opcaoCorreta->letra) {
      $corretas++;
      if (isset($materias[$mat])) $materias[$mat]++;
    } else {
      $erradas++;
      if (isset($materiasErradas[$mat])) $materiasErradas[$mat]++;
    }
  }

  return response()->json([
    'total_provas'         => $totalProvas,
    'total_questoes'       => $corretas + $erradas,
    'total_corretas_materia'=> $materias,
    'total_erradas_materia' => $materiasErradas,
  ]);
}
?>

Plano de Estudos — 8 Semanas

Roteiro
Estratégia: Cada semana tem ~5 horas de estudo. As primeiras semanas ensinam o PHP e MySQL. A partir da semana 4 você já estará construindo partes reais do projeto da prova. Na semana 7-8, você simula a prova completa cronometrada.
Semana 1PHP Básico

🐘 Fundamentos PHP

  • Instalar XAMPP + VS Code + extensões
  • Variáveis, tipos, operadores, strings
  • Arrays simples e associativos
  • If/else, foreach, while
  • Funções: criar, chamar, retornar
  • Exercício: Criar um arquivo PHP que receba um JSON e imprima os campos
Semana 2PHP + MySQL

🗄️ Banco de Dados com PDO

  • Criar banco no phpMyAdmin manualmente
  • Conectar PHP ao MySQL via PDO
  • Executar INSERT, SELECT, UPDATE
  • Prepared statements (segurança)
  • password_hash e password_verify
  • Exercício: Criar tabela de usuários e fazer o cadastro via PHP
Semana 3API REST Pura

🌐 API em PHP puro (sem framework)

  • Entender HTTP: métodos, status codes, headers
  • Ler body JSON, devolver JSON
  • Criar roteador simples
  • Implementar /registro e /login completos
  • Testar tudo com Postman
  • Exercício: API funcionando com registro, login e logout
Semana 4Laravel Intro

🚀 Primeiros passos com Laravel

  • Instalar Composer e criar projeto Laravel
  • Entender estrutura de pastas
  • Configurar .env com banco de dados
  • Criar primeira migration e model
  • Criar primeira rota e controller
  • Exercício: Recriar o /registro em Laravel
Semana 5Auth + Questões

🔐 Autenticação + Banco de Questões

  • Implementar login/logout com token no Laravel
  • Criar middleware de autenticação
  • Criar tabelas de questões e opções
  • Implementar endpoint de admin (banco de questões)
  • Cadastrar questões de teste via Postman
  • Exercício: Sistema completo de autenticação + CRUD de questões
Semana 6Provas

📝 Criação e Resposta de Provas

  • Criar tabelas de provas e respostas
  • Implementar endpoint de criação de prova (filtros)
  • Implementar responder questão
  • Implementar finalizar prova
  • Implementar consultar prova
  • Exercício: Fluxo completo: criar prova → responder → finalizar
Semana 7Métricas + Deploy

📊 Métricas + Deploy Linux

  • Implementar endpoint de métricas
  • Criar Seeder do admin padrão
  • Praticar deploy: FileZilla + PuTTY + Apache
  • Testar a API publicada no servidor
  • Exercício: Fazer deploy completo e testar todos os endpoints
Semana 8Simulado

⏱️ Simulado da Prova (4 horas)

  • Apagar o projeto e recomeçar do zero
  • Seguir o PDF da prova como se fosse o dia real
  • Usar o tempo: 4 horas cronometradas
  • Identificar o que ficou faltando
  • Repetir o simulado ao menos 1 vez
  • Meta: Entregar pelo menos 6 dos 9 endpoints funcionando
🎯 Prioridade de pontos na prova: Implemente nesta ordem — (1) Registro, (2) Login, (3) Logout, (4) Banco de Questões (admin), (5) Criar Prova, (6) Responder, (7) Finalizar, (8) Consultar, (9) Métricas. Os primeiros são mais simples e garantem pontos. Nunca entregue um projeto vazio.
